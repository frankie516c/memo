<h3>직접 만들어보기1</h3>

문제
```
const 기호 = "■";
for( ; ; )
{
  
}
// 출력 예시 : ■■■■■■■■■■ 
```

답
```
    const 기호 = "■";
    let 값 = "";
    for ( let i = 0 ; i < 10 ; i++ )
    {
       값 = 값 + 기호;
    }
     console.log(값);
```
- `값 = 값 + 기호;`를 `값 += 기호;`로 써도 같음

내가 헷갈린 부분들
- 값과 변수 / 기호를 구분하지 못함
  -  "■" 같은 문자 데이터를 숫자 변수 자리에 넣을 수 있다고 생각함
  -  자료형 개념(type) 이 아직 분리되지 않은 상태
- 몇 번 반복”과 “몇 개 출력”을 같은 것으로 착각
  - 사고 흐름: ■가 10개니까 for문으로 여러 번 console.log 하면 되겠지?
  - 실제 요구: 출력 1번
  - 헷갈린 개념: 반복 횟수는 데이터를 만들기 위한 과정
- 지금은 배열이 전혀 필요 없는 문제
  - 헷갈린 개념: 반복문 = 배열과 관련 있음 (아님. 관련 없음.)

<h3>직접 만들어보기2</h3>

문제
```
const 기호 = "■";
for( ; ; )
{
  
}
// 출력 예시 : 
// ■
// ■■
// ■■■
// ■■■■
// ■■■■■
// ■■■■■■
// ■■■■■■■
// ■■■■■■■■
// ■■■■■■■■■
// ■■■■■■■■■■
```

답
```
    const 기호 = "■";
    let 값 = "";
    for ( let i = 0 ; i < 10 ; i++ )
    {
       값 = 값 + 기호;
       console.log(값);
    }
```

내 오답
```
const 기호 = "■";
    let 값 = "";
    for ( let i = 0 ; i < 10 ; i++ )
    {
       console.log(기호);
       console.log("반복");
    }
    console.log(i);
```
내 실수
- `값 = 값 + 기호`를 넣지 않음
- `console.log("값")`을 넣지 않음
- `console.log("반복")`을 넣음
- `console.log("기호")`를 넣음

왜 이런 실수를 했는가?
- “반복하면 자동으로 결과가 쌓일 것”이라고 생각함
- 문자열은 직접 누적해서 저장해야 함
- for문 = 무조건 i < 10 같은 횟수 반복하는게 아님
- for문 = 초기화 / 조건 / 변화식을 자유롭게 설계하는 구조
