#라벨에 대하여
```HTML
<!DOCTYPE html>
<html lang="ko">
<head>
	<meta charset="UTF-8">
	<meta name="viewport" content="width=device-width, initial-scale=1.0">
	<link rel="icon" href="../../images/logo.png"/>
	<title>form</title>
</head>
<body>
	<h1>form</h1>
  <form action="result.html" method="get">
    <label for="userId">아이디 : </label>
    <input type="text" name="userId" id="userId">
    <input type="submit" value="인증요청">
  </form>
</body>
</html>
```
- input과 label이 하나의 쌍이다
    - **중요: label for="userId"랑 id="userId"를 동일하게 해야됨**
    - 같이 묶여있기 때문에 칸(input)이랑 아이디(label) 눌렀을 때 둘 다 칸안에 글자를 쓸 수 있음
    - 하나의 쌍이니까 div로 묶기도 함 (필수는 아님. 가독성 목적)
<div>
<label for="userId">아이디 : </label>
<input type="text" name="userId" id="userId">
</div>

```HTML

		  <label for="pwd">비밀번호: </label>

```
- name은 중복이 가능
- form에서 작성한 순서대로 F12 봤을 때 입력 순서가 지켜짐
- 주소에도 form에서 작성한 순서대로 창에 뜬다
- **백엔드 값이 잘 적용이 되는지 확인하기 위해서 F12의 Response 값을 확인해야 함**
- **request와 respond의 개념은 Python, 데이터분석 등에 다 적용되는 기본 개념**

#F12
- Elements: 코드가 보임(HTML, CSS only); script 부분 있는 ws
- Console: CSS가 보임
- Response: Javascript 보임
- ws: websocket - VSCode에서 값 바꾸면 실시간으로 화면 전환시키기 위해 라이브서버로 출력됨
  실시간으로 양방향 통신을 계속 할 수 있게 해주는 프로토콜
  HTTP는 요청->응답 구조이고 ws는 양방향인게 특징 (e.g., 채팅 서비스, 온라인 게임)
- status code에 101(switching protocols), 304(Not Modified) 등 확인 (404 Not Found 같은 애들)
