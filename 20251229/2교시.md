## 1️⃣ `App();`를 안 썼는데 왜 `App()`이 실행될까?

### App.js

```js
function App() {
  console.log("App()");
  return ( ... );
}
export default App;
```

여기서는 **함수 정의만 했고 호출은 안 했지**.
그런데도 실행되는 이유는 👉 **JSX가 대신 호출해 주기 때문**이야.

---

## 2️⃣ `export default App;` 는 어디로 가나?

### index.js (보통 이런 구조)

```js
import App from './App';

root.render(<App />);
```

여기서 중요한 포인트 👇

* `export default App`
  → **App 함수를 밖으로 내보낸 것**
* `import App from './App'`
  → **App 함수를 받아온 것**

하지만 아직도 **직접 호출은 안 했음**.

---

## 3️⃣ `<App />` 는 왜 `App()` 이 되는가?

### ✅ 이게 진짜 핵심

JSX에서 React는 이렇게 동작함:

```jsx
<App />
```

⬇️ 내부적으로 변환됨 (개념적으로)

```js
App();
```

정확히 말하면:

```js
React.createElement(App)
```

➡️ React는 **대문자로 시작하는 JSX 태그**를 보면

> “아, 이건 HTML 태그가 아니고
> **사용자 정의 컴포넌트(함수)**구나
> → 실행해야겠다”

라고 판단함.

그래서 자동으로 호출함.

---

## 4️⃣ 왜 하필 “대문자”일까?

### 🔹 JS / HTML 규칙

* HTML 태그는 **소문자**

  ```html
  <div></div>
  <span></span>
  ```

* JSX는 HTML처럼 보이지만 **JavaScript 문법**

* React는 구분이 필요함

그래서 React의 약속(규칙):

| JSX 문법            | 의미              |
| ----------------- | --------------- |
| `<div />`         | HTML 태그         |
| `<span />`        | HTML 태그         |
| `<App />`         | React 컴포넌트 (함수) |
| `<myComponent />` | ❌ HTML 태그로 오해   |

📌 **그래서 반드시 대문자로 시작해야 함**

네 문장을 기준으로 **조금 더 정확하게 정리해 보면** 👇

### ❌ 약간 애매한 표현

### ✅ 더 정확한 표현

> JSX에서 **대문자로 시작하는 태그는
> React가 HTML이 아닌 “컴포넌트 함수”로 인식하여 호출한다**

## 6️⃣ “HTML은 그려내는 거고, React는 함수를 부른다”는 말이 맞을까?

👉 **완전 맞음 (아주 중요한 개념)**

### HTML

```html
<div></div>
```

* 그냥 **모양**
* 로직 없음

### React JSX

```jsx
<App />
```

* 모양 ❌
* **함수 실행 결과를 화면에 그림**

즉:

```js
function App() {
  return <div>Hello</div>;
}
```

➡️ React는
“`App()`을 실행하고,
그 반환값(JSX)을 화면에 그려라”

## 7️⃣ 그래서 결론 한 줄 요약

> **JSX에서 대문자로 시작하는 태그는
> HTML이 아니라 ‘함수 컴포넌트’이며
> React가 자동으로 호출(`App()`)해 준다.**

## 1️⃣ 고쳐야 할(정확히 말하면 애매한) 표현들

### ❌ 원래 정리

> 리액트의 기본 함수형을 태그형식으로 만들면 JSX로 리턴해주면 함수 첫번째는 무조건 대문자로 사용하고 요소로 값을 전달할 수 있고

### 🔧 문제점

* “함수 첫번째” → ❌ 의미가 모호
* “태그형식으로 만들면” → ❌ JSX의 역할이 흐릿함
* “요소로 값을 전달” → ❌ props라는 개념이 빠짐

---

### ❌ 원래 정리

> 저렇게 `function Main({arr, list})` 처럼 실행 가능

### 🔧 문제점

* “실행”이라는 표현이 살짝 위험함
  → **직접 실행이 아니라 React가 호출**

---

## 2️⃣ 코드 기준으로 정확한 개념 정리

### 🔹 1. 함수형 컴포넌트의 조건

```js
function Main() { ... }
const Page1 = () => { ... }
```

React에서 **컴포넌트로 인식되려면**:

✅ 함수여야 함
✅ **이름이 대문자로 시작해야 함**
✅ JSX를 반환해야 함

---

### 🔹 2. JSX에서 컴포넌트는 “함수 호출”이다

```jsx
<Main />
```

React 내부 동작 개념:

```js
Main(props);
```

📌 그래서:

* 대문자 → 함수 컴포넌트
* 소문자 → HTML 태그

---

## 3️⃣ props 전달 부분에서 고쳐야 할 핵심 ❗

### ❌ 현재 코드

```jsx
<Main list={arr} />
```

```js
function Main({arr, list}) {
  console.log(arr, list);
}
```

### ⚠️ 문제

* `arr`는 **전달되지 않음**
* `list`만 전달됨

### 실제 값

```js
arr  === undefined
list === ["내용", "설명", "보조 내용"]
```

### 그래서 `arr?.map(...)`이 필요한 상태가 된 것임

---

### ✅ 올바른 방법 2가지

#### 방법 1️⃣ props 이름 통일 (추천)

```jsx
<Main arr={arr} />
```

```js
function Main({ arr }) {
  return (
    <main>
      {arr.map((v, i) => (
        <section key={i}>{v}</section>
      ))}
    </main>
  );
}
```

---

#### 방법 2️⃣ 전달한 이름에 맞게 사용

```js
function Main({ list }) {
  return (
    <main>
      {list.map((v, i) => (
        <section key={i}>{v}</section>
      ))}
    </main>
  );
}
```

---

## 4️⃣ 네 내용을 기반으로 “잘 정리된 설명문” 다시 써주기

### ✨ 정리본 (이걸로 외워도 됨)

> React에서 함수형 컴포넌트는
> **대문자로 시작하는 함수**로 정의하며
> JSX를 반환한다.
>
> JSX에서 `<컴포넌트 />` 형태로 사용하면
> React는 이를 HTML 태그가 아닌
> **함수 컴포넌트로 인식하여 자동으로 호출한다.**
>
> 컴포넌트에 전달하는 값은
> **속성 형태로 전달되며(props)**
> 함수의 매개변수에서 객체 구조 분해로 받을 수 있다.
>
> 예를 들어
> `<Main arr={data} />` 로 전달하면
> `function Main({ arr }) {}` 형태로 받을 수 있다.

## 5️⃣ 네 코드에서 중요한 개념 체크리스트 ✅

✔ `Page1`, `Page2`
→ JSX를 리턴하는 **화면용 컴포넌트**

✔ `Main({ arr })`
→ props 구조 분해
→ React가 호출

✔ `arr.map(...)`
→ 데이터 기반 렌더링

✔ `<section key={i}>`
→ 리스트 렌더링 시 key 필수

✔ `<>...</>`
→ Fragment (불필요한 div 방지)
